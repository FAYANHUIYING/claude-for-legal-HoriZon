# /deal-debrief 签约复盘

审查口径（playbook）最大的敌人是腐烂：合伙人实际接受的立场早就变了，文档里还是
旧的。本技能在记忆最新鲜的时候抓住每一次偏离——**不评判让步对不对，只记录它发生
了、为什么发生**。判断交给数据：同一条款反复让步，说明该改的是口径（由
`playbook-monitor` 代理提案）；偶发让步，说明只是个案。

## 前置

读取 `~/.claude/plugins/config/claude-for-legal-horizon/commercial-contract/CLAUDE.md`。
未配置或仍含 `[PLACEHOLDER]` → 提示先跑 `/commercial-contract:cold-start-interview`。
**playbook 尚不存在或为空时照常运行**——偏离日志可以先于成文口径积累（此时
"偏离"记为"无既定口径的让步"，恰好是未来 playbook 最该写的条款）。

偏离日志：`~/.claude/plugins/config/claude-for-legal-horizon/commercial-contract/deviation-log.yaml`

## 工作流

### 第 1 步 —— 找到要复盘的合同

按优先级：
1. 用户点名的合同/事项；
2. 事项工作区启用时，扫描各事项 `history.md` 近 7 天内含"签署/定稿/成交"的记录；
3. 都没有就直接问："上周签了哪些合同？逐个报名字即可，或说'没有'。"

零签约 → 一行报平安（"本周无签约，无偏离可记"），退出。

### 第 2 步 —— 逐合同问偏离（一轮 2-3 问，不滚屏）

对每份合同：

> 这份合同最终签署版里，有没有条款和我们的口径（或你的习惯立场）不一致？
> 常见位置：违约金上限、付款节奏、单方解除权、责任上限、担保、知识产权归属、
> 争议解决。逐条报：**哪个条款 → 我们的立场本来是什么 → 实际签成了什么**。

每条偏离追两个问题：
- **为什么让**？（对方强势/行业惯例/客户体量值得/换取了其他条款/时间压力——
  一句话即可，这是未来提案最值钱的字段）
- **谁批的**？（承办自决/团队负责人/合伙人——对照上报链，越级让步单独标记）

**有终版合同文件时优先读文件**比对 playbook 逐条核，而不是靠回忆——记忆会美化
让步。比对发现用户没报的偏离，列出来问"这处也算吗？"

### 第 3 步 —— 写入偏离日志

```yaml
deviations:
  - date: [签署日 ISO]
    contract: "[合同名/事项 slug]"
    counterparty_type: "[行业 + 体量，如：物流·大型国企]"
    clause: "[规范化条款名，如：违约金上限]"     # 同义条款用同一名称，便于聚合
    playbook_position: "[口径立场原文，或'无既定口径']"
    agreed_position: "[实际签署立场]"
    direction: [让步 / 反向收紧 / 新情形]
    reason: "[一句话原因]"
    approved_by: "[承办 / 负责人 / 合伙人姓名]"
    logged_by: "[记录人]"
```

**条款名规范化是聚合的命门**：录入前先查日志里已有的 `clause` 取值，同义的
（"违约金封顶"="违约金上限"）合并用既有名称，不要让同一条款在统计里裂成三个名字。

### 第 4 步 —— 收尾

- 回显本次写入的条目数与累计：「本次记录 N 条偏离；『违约金上限』条款 12 个月内
  已累计 M 次。」
- 某条款累计数**已达阈值**（默认 5 次/12 个月，配置可调）时，提示：
  「已达提案阈值——运行 `playbook-monitor` 代理看更新建议，或
  `/commercial-contract:review-proposals` 处理待审提案。」
- 决策树照共享护栏（深挖某条偏离 / 触发监控 / 完事）。

## 护栏

- **只记录，不裁判。** 本技能不评价让步明智与否——那是数据攒够后 monitor 提案、
  律师审批时的事。
- 偏离涉及客户可识别信息时，`contract` 字段用事项 slug 或脱敏名——日志在用户
  配置路径，但仍按最小化原则记。
- 绝不删除日志条目；记错的追加更正条目并在 `notes` 注明。
