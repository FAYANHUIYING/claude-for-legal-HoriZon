# 合同审查

合同审查的基本逻辑是"先看整体、再抠细节"。一份合同的风险，既藏在缺失的条款里，也藏在每一句模糊的措辞中。本工作流把审查拆成五个前后衔接的阶段，每个阶段都有明确的产出，前一阶段的结论是后一阶段的输入。

核心思路可以概括为一句话：**结构上不漏项，条款上不放过，结论上不武断。**

逐条审查时，对每一条款并行做三道相互独立的检验，取其中最高风险作为该条款的等级：

```
对每个条款 C：
  合法性  → 是否触碰强制性规定，导致条款无效或被行政处罚
  准确性  → 是否存在歧义、前后矛盾、关键术语未定义
  可操作性 → 履约标准能否量化验证、权利行使条件是否清晰、违约责任能否落地
  条款风险 = max(三者中最高者) → 映射到 🔴 🟠 🟡 🔵
```

三道检验不互相抵消：合法性致命的条款，不会因为"措辞很清楚"而降级。整份合同的评价 = 结构完整性 + 各条款风险汇总。

全流程设三个确认节点，避免方向跑偏：第一阶段末（确认立场与尺度）、第二阶段末（确认合同类型）、第五阶段末（请用户反馈是否有遗漏）。

---

## 第一阶段：确认审查前提

没有立场的审查是无效审查。开始读条款之前，必须先确认三件事，缺一不可。

### 1. 代表立场

先问清楚：站在哪一方审查？立场决定了风险容忍度和下手力度。

- **单方委托（默认）**：代表甲方或乙方其中一方，按该方的利益取向审查。
- **多方或混合立场**：当合同涉及三方及以上当事人（如联合体、三方采购、"委托+担保"的复合结构），或关联公司之间签约时，"甲乙二分"不再够用。此时要主动追问：审查应以哪一方或哪几方为本位？是否存在"集团整体最优"目标（个别条款对某关联方不利、但全局更优）？是否需要在不同条款段落切换立场（如总包方对业主时是一种立场、对分包方时是另一种）？确认后在报告中注明"多方立场——见审查前提说明"。

若用户上传了合同却没说立场，**先问再审**。

### 2. 修改尺度

按用户的谈判地位选一档：

- **只守生存底线**：谈判地位极弱（如小企业面对大平台的格式合同）。此时做风险分级：
  - *底线条款（必须争）*——触碰强制性规定的，以及关乎企业存亡的核心条件（付款节奏不致资金链断裂、违约责任不致一次违约即满盘皆输、核心资产不流失）。改不动就建议不签。
  - *非底线条款（力争）*——不致命但明显不公的，用"建议优化"而非"要求修改"的口吻提出，对方更易接受；改不动则记录风险并提示履约注意事项。
  - *不可争条款（接受并预警）*——对方核心商业模式条款，接受，但出具书面风险提示，写清最坏后果，由委托方知情后决策。
  - *替代保护*——条款改不动时，转用补充协议、关键节点的邮件确认（书面形式依法包含电子邮件）、付款前要求对方书面确认履约承诺等手段实现等效保护。
- **只改重大风险**：谈判地位偏弱，只提强制性规定违反和根本性权利缺失。
- **对等调整**：地位相当，推动双方风险分配均衡。
- **适度争取**：地位较强，在合规前提下为己方多争取利益，但不可用力过猛到显失公平。

### 3. 输出模式

表格对比版 / 直接修改版 / 修订留痕版 / 清洁版 / 法律意见书。用户未指定时，默认按法律意见书格式输出。其中表格对比版采用"原条款｜风险点｜修改建议｜优先级"四列对照，便于内部流转。

### 4. 补充信息（尽量索取）

闭门审合同容易脱离实际。除立场/尺度/模式外，主动向用户索要以下信息，它们直接影响审查方向：

- **交易背景与目的**：为什么签这份合同，想达成什么？
- **双方强弱**：谁提供的格式文本，谁更急于促成？
- **核心关切**：最担心的是什么（付款安全 / 质量 / 知识产权 / 违约）？
- **关联文件**：意向书、会议纪要、框架协议、历史版本——核对是否有冲突约定。

用户暂时拿不出这些信息时，在报告里注明"基于合同文本本身审查，未结合交易背景"。

### 5. 审查深度（主动建议，用户确认）

审查投入应与合同价值、风险大小匹配。先从文本里提取页数、金额、合同类型、主体数量等客观指标，再据此建议深度：

| 深度 | 大致适用 | 范围 |
|------|---------|------|
| 🔵 快速扫描 | 金额很小、页数很少、且非高风险类型 | 只查底线条款（主体资格、担保/连带责任、管辖约定、违约金异常、强制性规定违反）+ 结构快速排查 |
| 🟡 标准审查 | 一般金额、常规类型，或用户要求 | 快速扫描全部 + 四大主题完整审查 + 逐条三维审查 + 谈判优先级矩阵 |
| 🔴 深度审查 | 金额较大、高风险类型、涉及多方，或用户要求 | 标准审查全部 + 专项清单全量 + 配套文件完整扫描 + 行业法规与官方示范文本对照 |

**强制升级**：以下类型不论页数金额，一律不得只做快速扫描——担保/保证合同、对赌/估值调整、竞业限制、股权转让/代持、建设工程合同，以及格式合同中出现"连带""保证""无限"等字样的。这类合同里"无限连带责任保证"几个字就足以让签字人倾家荡产，页数金额与法律风险并非线性关系。

三条铁则：①再快也不跳过底线条款，深度分级是"打几分"而非"做不做"；②高风险类型自动拦截升级；③审查结束时在报告"审查前提"里如实列出已覆盖与未覆盖的维度，让用户知情决策。

## 第二阶段：识别合同类型

### 穿透标题看实质

不要只看合同标题。很多合同名实不符（名为"合作协议"实为"借款合同"）。判断方法：看双方核心权利义务如何配置，对照各类典型合同的法律特征，确定其真实法律性质。若是混合合同（含多种典型合同要素），分别识别各部分适用的规则。

### 匹配审查重点，并做相关性过滤

识别类型后，调用该类合同的常见风险维度。但**不能机械套用**——不同合同的核心风险维度不同，逐项自问"这个维度和本合同的实质交易是否相关"，不相关的标注"不适用"、不在报告里展开。例如设计类工程合同不涉及施工现场，就不必套用农民工工资、安全事故责任等条款。在报告中注明过滤结果。

### 扫描配套文件

很多合同"主文薄、义务厚"——大量实体义务通过"见附件X""按某某制度执行""遵守某某手册"引用到外部。开始审查时就系统扫描：列出全部被引用的外部文件，逐一标注"是否已提供 / 是否双方签字确认 / 是否可由一方单方修改"。凡出现"未提供""未确认""可单方修改"的，在结构校验中作为独立风险标注。常见高发场景：物业合同引用的业主手册与管理规约、采购合同引用的验收标准、劳动合同引用的员工手册。

### 核查前置程序

有些交易，法律法规要求签约前先履行前置程序，否则影响效力或引发责任。结合主体、标的、交易方式判断是否需要，例如：国有资产转让需评估并报批、公司对外提供担保需有效的股东会/董事会决议、特定经营权转让需主管部门审批等。报告中标注"前置程序：是否需要→是否已完成"，需要但未完成的作为致命风险。

### 类型确认节点

进入下一阶段前，向用户简短确认："识别本合同为【XX类型】，按对应重点审查，是否正确？"快速通过即可，用户纠正则重新匹配。

## 第三阶段：校验结构完整性

用"四大主题"框架对合同做整体扫描，识别结构性漏项：

| 主题 | 核心追问 | 子维度 |
|------|---------|--------|
| **交易主体** | 各方是否有能力、有资格完成本交易？ | 主体身份、资质许可、地址与送达、第三人履约 |
| **交易内容** | 标的、价款、权利归属是否明确完整？ | 标的规格/数量/质量、价款构成、知识产权归属、术语定义 |
| **交易方法** | 履约、交付、验收、结算每一步是否有可操作的标准？ | 时间地点批次、交接运输投保、验收程序、付款结算、通知送达 |
| **问题处理** | 出问题时各方的责任与救济路径是否清晰？ | 违约判定与责任、不可抗力/情势变更、解除条件、争议解决、生效失效 |

同时对照法律规定的合同一般必备条款做底线校验：当事人、标的、数量、质量、价款或报酬、履行期限地点方式、违约责任、争议解决方法。

### 持续性合同的叠加维度

若为周期较长（约一年以上）的持续性服务合同（物业、维保、外包、SaaS 等），在四大主题之上再叠加五个维度：

| 维度 | 要点 |
|------|------|
| 服务质量可持续 | 标准能否量化验证，有无逐年退化风险，有无定期评估机制 |
| 中期调整机制 | 价格是否锁定，调价触发条件是否客观（如挂钩公开指数）而非主观 |
| 退出与交接 | 解约程序是否明确，单方解约权归属，交接义务（资料/客户信息/预付款清算）是否约定 |
| 连续性保障 | 服务中断有无应急，更换服务商的过渡如何保障 |
| 单方变更限制 | 一方能否单方修改服务标准或管理规则，新规则对旧约定有无溯及 |

未约定上述内容的，在结构校验中标注"持续性维度缺失"。

**产出**：列出缺失的条款维度，并标注缺失可能导致的法律后果。

## 第四阶段：逐条款风险审查

### 审查纪律：不许一次过

找到一个高风险条款后不能就此收工。每轮逐条审查结束，强制做一次系统回查：对照本合同的核心风险维度逐项核对；确认所有被引用的外部文件都已在前提中标注；把分散在多处、却指向同一事项的条款（如报价、知识产权、违约责任、终止处理）串起来看联动风险。在报告中注明"一轮发现 X 项→回查补充 Y 项"。

### 受监管行业：先查法规与示范文本

若合同涉及受监管行业（物业、建设工程、金融借贷、劳动用工、医疗药品、房地产等），进入条款审查前先检索该行业现行核心法规与司法解释，并平行检索是否有主管部门发布的官方示范文本。对照示范文本审查往往比对照法条更直接：把本合同与示范文本逐项比对，标注"本合同缺失但示范文本要求"的条款，作为完整性缺陷的依据。报告中注明已检索的法规版本/修订时间。

### 三维审查

**合法性**——是否违反效力性强制规定（致条款无效）或管理性强制规定（致行政处罚）；区分强制性与任意性规定；注意司法解释与法律的层级差异、新旧法衔接与时效。

**准确性**——是否存在歧义（同一条款两种以上合理解释）；条款之间是否矛盾；关键术语是否定义；数量、期限、比例是否明确可执行。

**可操作性**——履约标准能否量化验证；权利行使的条件与程序是否明确；违约责任是否可执行（而非空泛的"承担法律责任"）。

### 两个叠加视角

- **抓重点**：先标出"核心利益条款"（卖方看付款安全与交付条件、买方看质量保障与验收权），对它们逐字逐句；辅助性格式条款快速通过。
- **看可行**：约定的履行时间是否现实（考虑审批办结、对方履约能力）；对方注册资本与交易规模是否相当、历史履约如何；对关键节点做"如果对方无法按时履行，我方有何替代方案"的压力测试。

### 引用法条的规矩

- 精确到条文号和生效时间，不写"根据相关规定"。
- 引用前先核验该条文是否现行有效。
- 一律标注"请核实"，并指明权威验证渠道。
- 检索不到时如实说"未检索到"，**绝不捏造条文**。

## 第五阶段：风险定级与报告

实质分析在前四阶段已经完成，本阶段是把分散结论汇总呈现。

### 风险定级与分类

**等级**：🔴 致命（效力瑕疵/违法）｜🟠 重大（核心权利缺失或严重失衡）｜🟡 中等（条款模糊/操作困难）｜🔵 轻微（表述优化）。

**问题类型**（便于按类批量修订）：

| 类型 | 含义 |
|------|------|
| 缺失 | 应约定而完全遗漏（无验收标准、无知识产权归属、无保密条款） |
| 违法 | 内容违反强制性规定（排除人身损害赔偿、约定违反专属管辖） |
| 模糊 | 用语含混、多义或不可量化（"合理期限""及时通知""重大违约"） |
| 失衡 | 权利义务显著不利于审查方（单方任意解除、风险全转嫁、无限连带） |
| 冲突 | 不同条款对同一事项相互矛盾（付款时间前后不一致） |

每个风险还要标注**引爆条件**（什么情况下风险实际爆发）和**法律依据**（条文号+生效时间，标注"请核实"）。

### 多路径修改建议

每个风险给出多条路径（至少两条），让律师按具体场景选：

- **路径A（核心修改）**：有谈判筹码、可直接改。给出替代条款全文、法律依据、修改后可能产生的新风险、适用条件。
- **路径B（折中/最小修改）**：筹码有限，只能微调。在原条款上做最小改动，标注保留的风险和让步边界。
- **路径C（替代控制）**：条款完全改不动。不动条款本身，转用书面锁定事实、改造履约流程、引入第三方对冲、固化证据、分批操作等手段实现等效风险控制。

三条路径不是"好→中→差"的降级，而是不同谈判场景下各有最优。**不给唯一结论或"推荐方案"**，把最终决策权留给法律人。

### 谈判优先级矩阵

按"法律重要性 × 谈判可行性"两轴把风险分到四象限：

| 象限 | 含义 | 策略 |
|------|------|------|
| 🔴 必争 | 不改有实质法律风险，且己方有筹码 | 集中火力、据理力争、不做交换 |
| 🟠 观察 | 风险高但对方难让步 | 迂回管控：履约监督、补充协议、备忘录 |
| 🟡 筹码 | 己方不太在意、对方可能在意 | 主动让渡，换对方在必争项上妥协 |
| 🔵 放弃 | 影响有限且难推动 | 不提，节省谈判精力 |

### 报告结构（强制遵循）

章节顺序、字段名称不得增删。语言正式，使用法律书面语，禁止口语化、比喻、网络用语。四列速查表是导航、逐条详细分析是核心，两者缺一不可。

```markdown
# [合同名称] 审查报告

## 审查前提
- **代表立场**：[甲方/乙方/中立/多方]
- **修改尺度**：[只守底线/只改重大风险/对等调整/适度争取]
- **合同类型识别**：[法律性质认定]
- **审查深度**：[快速/标准/深度]｜已覆盖：[…]｜未覆盖：[…及升级路径]
- **补充信息**：[交易背景提供情况 / 基于合同文本本身审查]
```

## 十条审查准则

1. **审查前先问立场**——没有立场的审查是无效审查。
2. **穿透标题看实质**——合同标题不等于法律性质。
3. **法条必须检索验证**——不凭记忆引用，引用即标注"请核实"，检索不到就如实说，绝不捏造。
4. **风险要排序、要标引爆条件**——做风险地图，不做风险罗列。
5. **给多方案、不给唯一结论**——决策权留给法律人。
6. **报告是行动手册不是教科书**——每个判断附推理与依据，可逆向验证。
7. **区分法定要求与实务建议**——前者有约束力，后者供决策参考。
8. **不放过底线条款**——再快的审查也要查主体资格、担保连带、管辖、违约金异常、强制性规定违反。
9. **高风险类型自动升级**——担保、对赌、竞业限制等不因页数金额小而降低深度。
10. **如实告知未覆盖项**——让用户在知情的前提下决策。
