# 民事起诉状撰写

本技能引导你把零散的案件信息整理为一份结构规范、要素齐全、逻辑自洽的民事起诉状。
内置标准模板与逐步工作流，确保产出符合《民事诉讼法》及相关规定对起诉状的要求。

**重要提示**：本技能仅提供文书撰写辅助，所生成内容为草稿，不构成法律意见。具体案件
应由具备相应资质的执业人员核校后使用；当事人主体信息、金额、管辖及法律条文均须经独立
核实方可定稿。

---

## 工作流总览

1. **要素采集** — 提取当事人与案件事实，识别信息缺口
2. **法律梳理** — 确定案由、诉讼请求与法律依据
3. **分段撰写** — 依模板逐部分生成正文
4. **一致性自检** — 交叉核对各部分是否吻合
5. **定稿输出** — 标注待核实项后交付

---

## 第一步：要素采集

用户提供的材料可能是口头叙述、聊天记录、合同或票据。先从中提取结构化要素，再逐项核对是否齐全。

### 要素清单

| 类别 | 需采集内容 | 是否必备 |
|------|------------|----------|
| 原告 | 自然人：姓名、性别、出生日期、民族、住所、身份证号；法人/组织：全称、住所、统一社会信用代码、法定代表人 | 必备 |
| 被告 | 同原告项；如为自然人另记联系方式与可送达地址 | 必备 |
| 第三人 | 同被告格式（若涉及） | 视情形 |
| 案件经过 | 事件起因、关键时间节点、双方行为 | 必备 |
| 争议行为 | 被告违反何种约定义务或实施何种侵权行为 | 必备 |
| 损害结果 | 原告由此产生的具体损失及计算方式 | 必备 |
| 证据线索 | 已掌握证据的名称与形式 | 尽量提供 |
| 拟诉法院 | 有管辖权的法院（如已知） | 尽量提供 |

### 信息缺口的处理原则

- **硬信息一律不臆造**：当事人全称、统一社会信用代码、身份证号、争议金额、管辖法院等
  必须来自用户提供或可核实的来源。缺失时以占位符标注，例如 `{原告统一社会信用代码：待核实}`，
  并提示用户补充，切勿凭推测填写。
- 事实表述含糊处，就关键细节向用户确认后再落笔。
- 用户未提及的要素，逐项询问而非默认或省略。

---

## 第二步：法律梳理

在事实基本清晰后，做初步法律定位，为撰写诉讼请求与理由打底。

1. **确定案由**：依现行案由规定，选取与事实最贴合的案由。案由牵动法律适用与请求方向，
   宜先定准。
2. **拟定诉讼请求**：将诉求逐项拆分，每项均须具体、可执行，避免"赔偿损失若干"之类模糊表述。
3. **匹配法律依据**：为每项请求找到支撑性法条。**法条不得凭记忆引用**——引用前先核实其
   是否存在、是否现行有效，必要时检索确认；无法确认者标注"法条待核实"。
4. **判断管辖**：依案件性质与当事人情况推断有管辖权的法院，并提示用户核对级别管辖与
   地域管辖（含约定管辖、专属管辖等特别规则）。

---

## 第三步：分段撰写

参照本文末尾的"起诉状标准模板"，按以下顺序逐部分生成正文。

### 3.1 当事人信息
- 区分自然人与法人/其他组织，分别采用对应写法。
- 多名被告时以"被告一、被告二……"编号，全文称谓保持统一。
- 主体硬信息缺失处保留占位符，不得编造。

### 3.2 诉讼请求
- 排列顺序：确认之诉 → 给付之诉 → 程序性请求（如诉讼费负担）。
- 每项独立编号，前各项以分号收尾，末项用句号。
- 涉款项须写明币种与确切数额，并与"事实与理由"中的金额一致。

### 3.3 事实与理由
此为起诉状核心，建议分层展开：
1. **法律关系铺垫**：交代原被告之间的基础关系（合同、侵权或其他）。
2. **事实叙述**：按时间顺序客观陈述经过，关键事实括注对应证据。
3. **责任分析**：逐项指出被告的违约或侵权行为，每项对应具体条款或法定义务。
4. **损害说明**：阐明损失及计算口径，与诉讼请求金额对应。
5. **法律论证**：援引相关法条论证诉求成立（法条须经核实）。
6. **收束**：以"综上所述"概括，请求法院依法支持诉请。

撰写要诀：请求与事实一一对应、保持客观中立、引用务求准确。

### 3.4 此致法院与落款
- 法院名称使用规范全称。
- 具状人署名，起诉日期一般填提交之日。

### 3.5 证据目录
- 按证明对象分组，逐条列明证据名称与证明目的。
- 证明目的应与对应的诉讼请求相呼应。

---

## 第四步：一致性自检

成稿后逐项交叉核对：

| 检查项 | 核对内容 |
|--------|----------|
| 称谓统一 | 各部分当事人称谓是否前后一致 |
| 请求—事实 | 每项请求是否均有对应事实段落支撑 |
| 请求—依据 | 每项请求是否均有法律依据 |
| 金额吻合 | 事实理由与诉讼请求中的金额是否一致 |
| 证据覆盖 | 关键事实是否均有证据指向 |
| 格式规范 | 各部分是否符合模板要求 |
| 法条有效 | 援引法条是否存在且现行有效 |

发现问题直接修正，并向用户说明改动之处。

---

## 第五步：定稿输出

将起诉状整理为完整文本交付，并附一段"待核实清单"，提示用户：
- 仍为占位符、需补充的主体硬信息；
- 建议进一步核实的事项（法条时效、管辖法院、金额计算等）；
- 起诉状含当事人隐私信息，请妥善保管。

---

## 示例

**示例一（信息较完整）**：用户给出借贷双方身份、借款与逾期事实。
执行：提取要素 → 定案由（民间借贷纠纷）→ 拟请求（返还本金、利息、诉讼费）→ 套模板成文 → 自检输出，
对身份证号等未核实项保留占位符提示。

**示例二（信息含糊）**：用户称"被公司辞退，想告公司违法解除"。
执行：识别缺口（公司全称、工作年限、工资标准、解除时间、有无书面通知等）→ 逐项追问 →
信息齐备后定案由（劳动争议）→ 撰写 → 自检输出。

## 起诉状标准模板

> 占位符 `{……}` 处填入经核实的信息；无法核实者保留占位并在交付时提示用户。

```
民事起诉状

原告：{姓名/名称}，{性别}，{出生日期}，{民族}，
住{住所地}，
身份证号/统一社会信用代码：{……：待核实}，
联系电话：{……}。
（法人或其他组织另列：法定代表人/负责人 {姓名}，职务 {……}。）

被告：{姓名/名称}，{性别}，{出生日期}，{民族}，
住{住所地}，
身份证号/统一社会信用代码：{……：待核实}，
联系电话：{……}。
（多被告时依"被告一、被告二……"分列。）

第三人：{……}（如涉及，格式同被告；无则删除本行）

诉讼请求

一、判令被告{具体给付/确认事项，写明币种与数额：……}；
二、判令被告{第二项请求：……}；
……
{末项}、本案诉讼费用由被告承担。

事实与理由

（一）基础法律关系
{交代原被告之间的合同/侵权或其他法律关系的形成。}

（二）案件事实
{按时间顺序客观陈述经过，关键事实括注证据，如（见证据X）。}

（三）被告的违约/侵权
{逐项指出被告行为及其所违反的约定条款或法定义务。}

（四）损害后果
{说明原告损失及计算方式，与诉讼请求金额对应。}

（五）法律依据
{援引相关法条论证诉请成立——法条须经核实，未核实者标注"待核实"。}

综上所述，被告的行为已构成{违约/侵权/……}，依法应承担相应责任。
为维护原告合法权益，特依据《中华人民共和国民事诉讼法》相关规定向贵院提起诉讼，
请求依法支持原告的全部诉讼请求。

此致

{受诉法院全称：待核实}

具状人（签名/盖章）：{原告}
{年}年{月}月{日}日

附：
1. 本起诉状副本{份数}份；
2. 证据目录及证据材料{份数}份；
3. 原告主体资格证明{……}。

证据目录

证明对象一：{……}
  证据1：{名称}——证明目的：{……}
  证据2：{名称}——证明目的：{……}
证明对象二：{……}
  证据3：{名称}——证明目的：{……}
```
