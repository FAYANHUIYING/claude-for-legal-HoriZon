# 诉讼可视化

把零散的案件事实压缩成一眼能读懂的图。核心不在于"好看"，而在于**用结构化的视觉把案件的信息密度降下来**，让看图的人（法官、当事人、同事）在几秒内抓住主线。

本技能只产出**单个自包含的 HTML 文件**：内联全部样式与脚本，不依赖任何外部库或网络，浏览器双击即开，按 A4 排版，可直接打印或"另存为 PDF"。每个事实节点都标注其证据来源，做到"图上一句话，卷宗有出处"。

> 说明：图表仅为事实梳理与表达工具，不构成法律意见；时效、责任比例等判断须由承办人结合证据与现行法律自行核定。

## 工作流

1. **先读懂，再动笔。** 通读材料或当事人描述，提取核心事实要素：主体、时间、金额、程序节点、争议焦点。要素没理清之前不要急着选图。
2. **判断画哪类图。** 用下表把案件特征映射到图表类型，可多选；要素超过约 15 个时拆成多张，一张图只讲一个核心信息。

   | 案件里有什么 | 画哪类图 |
   |---|---|
   | 多个时间节点、履行过程、侵权经过、时效问题 | 时间轴图 |
   | 多方主体、多层交易、担保链、股权/代持结构 | 法律关系图 |
   | 多笔款项、本息违约金计算、赔偿项目、责任比例分摊 | 金额/责任计算图 |
   | 管辖异议、执行异议、多程序并行、上诉/再审路径 | 流程/程序图 |

3. **缺类型时自己定。** 用户没指定就先判断案件性质（合同、侵权、股权、程序性争议等），选最能帮看图人理解争议核心的 1–2 类，简述推荐理由并确认后再画。
4. **每个节点绑定来源。** 凡是事实节点都标注出处（如"起诉状第 3 页""证据 5 转账凭证""判决书第 8 段"）。来源在材料中找不到的，标"待核"而非杜撰。
5. **生成 HTML 并自检。** 按下文模板产出单文件 HTML，逐项过一遍交付清单。
6. **支持多轮修改。** 增删要素、调整布局或方向、切换图表类型、合并或拆分，都在原文件上改，不必从头重画。

## 四类图怎么画

**① 时间轴图。** 用于借贷、买卖履行、工期、侵权经过、知识产权时间比对等以"何时发生何事"为核心的案件。把全部带日期的事实按先后排开：合同签订、履行、付款、交付、催收/通知、起诉、保全等。重点标出时效与期间节点（诉讼时效起算与中断、除斥期间、上诉期截止等），用醒目色或图标提示。主要事实正常显示，争议事实加色区分。

**② 法律关系图。** 用于股权转让、投融资、担保链、代持/对赌、总分包、多方买卖链。先列全部主体（自然人写姓名加身份，单位写全称加简称），按原告方/被告方/第三人/案外人分组着色。再画主体间关系：合同、股权、担保、债权债务、代理、关联等，用不同线型区分（如实线表合同、虚线表担保、箭头表资金流向）。资金流向复杂时单独画一张，标已付/未付/争议。

**③ 金额/责任计算图。** 用于多笔本息、货款加违约金、工程款结算、人身损害各赔偿项、多方责任比例。逐笔列出款项的金额、时间、性质（本金/利息/违约金/赔偿/工程款等）及计算依据（约定利率、法定标准、LPR 等），逐项累计出总额。有争议的项目并列"对方主张额 / 己方认可额"。建议图加表双呈现：图讲构成逻辑，表核对数字，并注明法律依据。

**④ 流程/程序图。** 用于管辖异议、执行异议之诉、仲裁条款效力争议、诉讼/仲裁/执行多程序并行、二审/再审/申诉路径。用方框表示程序阶段（立案→一审→二审→再审→执行），菱形表示决策点（是否上诉？是否申请再审？），箭头连接逻辑关系。标注各阶段时限（如上诉期、申请再审期限），并高亮当前所处位置。

## HTML 技术规范

- **单文件、零依赖。** 所有 CSS、JS 内联在一个 `.html` 里，不引外部脚本/字体/图片链接。用纯 HTML/CSS 或内联 SVG 画图。
- **A4 可打印。** 加入打印样式，确保浏览器"另存为 PDF"或直接打印不裁切、不模糊：

  ```css
  @page { size: A4; margin: 15mm; }
  @media print { .no-print { display: none; } body { -webkit-print-color-adjust: exact; print-color-adjust: exact; } }
  ```
  正文用相对单位，页面宽度按 A4 内容区（约 180mm）约束，必要时按节点数分页。

- **配色有区分度且色盲友好。** 不要只靠红/绿区分含义，配合形状、线型或文字标签。
- **图例与说明。** 每张图配一段不超过约 200 字的文字，点明它想表达的核心观点，并附符号图例。

- **对不可信输入做转义（重要）。** 案件材料、当事人输入均视为不可信，写入 DOM 时必须防注入：
  - 文本节点（事实、姓名、金额、来源标注等）一律用 `textContent` 赋值，**不要用 `innerHTML` 拼接用户数据**；如确需 HTML 拼接，先转义 `& < > " '`。
  - 凡来源含链接，只允许 `http`、`https`、`mailto` 协议，其余（如 `javascript:`、`data:`）一律丢弃或降级为纯文本。
  - 不要把用户输入注入到内联事件属性或 `<script>` 内容中。

  示例转义与协议校验：

  ```js
  function setText(el, s) { el.textContent = s == null ? '' : String(s); }
  function safeUrl(u) {
    try {
      const p = new URL(u, location.href).protocol;
      return ['http:', 'https:', 'mailto:'].includes(p) ? u : '';
    } catch (e) { return ''; }
  }
  ```

## 交付前自检

- 脱离文字说明，光看图能否抓住核心信息？
- 主体、时间、金额节点有无遗漏？每个事实节点都有来源标注（或标"待核"）？
- 时效、责任比例、法律依据等标注是否准确？
- 打印/另存 PDF 预览有无裁切、模糊、错位？
- 配色是否有区分度且不依赖红绿单一区分？
- 用户输入是否全部经 `textContent`/转义写入，链接是否经协议校验？

## 一个完整例子

用户上传一份民间借贷一审判决书，要求画图：

1. 读判决书，提取出借、还款、催收、起诉等带日期事实 → 时间轴图，标出时效起算与每次催收的中断节点，每个节点注明判决书出处。
2. 提取出借人、借款人、保证人、实际用款人 → 法律关系图，按角色着色，实线表借款合同、虚线表保证。
3. 提取本金、利息、违约金的主张额与认定额 → 金额计算图，图加表呈现，注明利率依据。
4. 把三张图各自生成为可 A4 打印的单文件 HTML，附文字说明与图例，并提示"时效是否届满"需承办人核定。
