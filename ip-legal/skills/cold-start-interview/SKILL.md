# /cold-start-interview

1. 检查 `~/.claude/plugins/config/claude-for-legal-horizon/ip-legal/CLAUDE.md`。若带 `--check-integrations`，跳过访谈——只重跑第 0 部分的"接了什么？"检测，并在该配置路径重写 `## 可用集成` 表。探测时：只有当某 MCP 工具调用确实成功，才报告 ✓；已配置但未测试的连接器标 ⚪ 并附一行确认方法。**绝不仅凭 `.mcp.json` 声明就报告 ✓**——那会误导用户以为接好了，其实没有。
2. 运行下面的访谈：先第 0 部分（角色 + 执业设置 + 集成），再领域专属问题。
3. 索取种子文件（见下）。
4. 写入 `~/.claude/plugins/config/claude-for-legal-horizon/ip-legal/CLAUDE.md`，按需创建上级目录。

## 冷启动检查

读取 `~/.claude/plugins/config/claude-for-legal-horizon/ip-legal/CLAUDE.md`：
- **不存在** → 开始访谈。
- **包含 `<!-- SETUP PAUSED AT: -->`** → 向用户问好，提议从该小节继续。
- **包含 `[PLACEHOLDER]` 标记但无暂停注释** → 模板从未完成；提议重新开始，或从占位符起始处继续。
- **已填充（无占位符、无暂停注释）** → 已配置；除非带 `--redo`，否则跳过。

模板结构在 `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md`——以它作为分节脚手架。把填好的实务画像写入配置路径，按需创建上级目录。

## 检查共享的所务画像

查找 `~/.claude/plugins/config/claude-for-legal-horizon/company-profile.md`。

- **若存在：** 读取它。一行确认："你是 [姓名]，[执业设置]，就职于 [律所/公司]，在 [管辖地] 执业。对吗？（或说'更新'来修改共享画像。）" 确认后跳过所务问题，直接进入本插件专属问题。
- **若不存在：** 你将是该用户配置的第一个插件。问所务问题（所名/业务/规模/管辖地/风险姿态/上报联系人），按本插件 `references/company-profile-template.md` 模板写入共享画像，并告诉用户："我已保存你的所务画像——其他法律插件会读取它并跳过这些问题。"

## 访谈开始前

以"先分叉"的开场白起头，3-4 行短句，先问速启还是完整：

> **`ip-legal` 面向处理商标、专利、著作权、商业秘密事务的人**——咨询体检、权利要求分析、可注册性初筛、侵权判断。
>
> **2 分钟**：完成你的角色、执业设置、常代理立场（权利人/被诉方）和集成检测。 **15 分钟**：再加上组合规模、常见技术/商品领域、代理机构对接和过往分析意见的格式提取。
>
> 速启还是完整？（随时可用 `/ip-legal:cold-start-interview --redo` 升级。）

**速启路径：** 只问第 0 部分和最关键的 1-2 个领域问题，其余以 `[DEFAULT]` 标记写入配置，并收尾说明：当某技能输出感觉不对劲时，通常就是该调的某个默认值——它会告诉你是哪个。

**完整路径：** 走完下面全部小节。

设置只从你的回答和你明确上传的文档构建画像——不读个人 Claude 历史、其他对话或主目录下的 CLAUDE.md。若对话上下文中已可见相关信息（如先前提到的所名），先问"要用它吗？"再使用。

## 访谈节奏

- **假设答案已经写在某处。** 凡超过一句话的内容，默认问法是"贴个链接或文档，或给我简短版"——别逼用户把已写好的东西重打一遍。
- **一轮不超过 2-3 个可作答的提问**（子项也算）。检验：用户能否不滚屏答完？
- **暂停与续接。** 开头告诉用户：说"暂停"即保存进度——写部分配置并在顶部加 `<!-- SETUP PAUSED AT: [小节名] -->`，未答字段标 `[PENDING]`。重跑时从停处继续，不重问已答的。
- **绝不写带静默缺口的配置。** 写入前回看：列出被跳过的问题，问"现在补几个，还是留作占位符？" 每个占位符都应是用户主动选择跳过的。
- **用户陈述的法律事实随到随核。** 用户给出的条号、期限、阈值、管辖地与你所知冲突时，写入配置前先说出来：「你说 X；我的理解是 Y——确认哪个写进画像？`[前提已标记—需核实]`」错误事实进了 CLAUDE.md 会传播进未来每份输出。

## 第 0 部分：谁在用，以及接了什么

#### 谁在用？

> 谁会日常使用本插件？（这决定每份输出的工作成果页眉。）
>
> 1. **律师/法律专业人员** → 页眉 `保密 — 律师工作成果 — 宏志律师事务所 · 依律师指导制作`
> 2. **非律师但有律师对接** 3. **非律师且无对接** → 页眉 `研究笔记 — 非法律意见 — 据以行事前须经执业律师审查`

若答案是 2 或 3，说一次：输出将表述为供律师审查的研究而非定论；在有法律后果的步骤（签署、发函、立案、回复机关）前会暂停并提示律师审查。这不是免责声明，是工具擅长什么（检索/组织/结构）与持牌律师对具体情形的判断之间的分界。

#### 执业设置

> 独立执业/小所（无层级） | 中大型律所 | 企业内部法务 | 政府/法律援助/公益 | 都不是（用你自己的话描述，我据实建画像而不硬塞框）

独立/小所 → 上报问题改问"何时请外聘律师或同事给第二意见"；中大型所/内部法务 → 问审批链与签字人；公益 → 问督导结构。

#### 接了什么？

> 本插件可对接：法律检索（元典/北大法宝）、飞书/IM、文档存储。让我检查你配了哪些——需要它们的功能会工作，没有的优雅回退到手工，而非静默失败。

**检查实际接通的，而非配置了的**：能测试就调一个简单 MCP 工具，成功才报 ✓；无从探测报 ⚪"已配置未验证"；未找到报 ✗ 并附接入方法与回退说明。日后接上了，重跑 `/ip-legal:cold-start-interview --check-integrations`。

把以上写入配置的 `## 使用者角色`、`## 可用集成` 小节。

## 领域专属问题

### 第 1 部分：知产画像

> 1. **常代理立场**：权利人维权为主，还是被诉方应对为主？（侵权分析的攻防方向相反。）
> 2. **领域集中度**：商标为主、专利为主还是综合？常见商品/技术领域（尼斯分类哪些类、IPC 哪些部）？
> 3. **代理机构对接**：申请类业务（商标注册/专利申请）是否外发代理机构？本插件不替代申请代理，只做分析与初筛。

## 种子文件

> 1. **权利清单**（商标/专利组合，如有）。
> 2. **1-2 份过往侵权分析意见**（脱敏）。我提取要件分析的写法与结论分级习惯。
>
> 粘贴、给路径，或"暂时跳过"（标 [LIMITED DATA]）。

## 撰写实务画像

按 `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md` 的模板结构，把填好的画像写入 `~/.claude/plugins/config/claude-for-legal-horizon/ip-legal/CLAUDE.md`。凡基于少于足量文档构建的小节标 `[LIMITED DATA — 已审阅 N 份文档]`。

## 写完之后

**展示本插件能做什么**（具体清单，不是笼统模板）：

> - **知产综合咨询/体检**——`/ip-legal:ip-advisor`
> - **专利权利要求与侵权/FTO 分析**——`/ip-legal:patent`
> - **商标类别规划与可注册性初筛**——`/ip-legal:trademark`
> - **商标侵权与混淆可能性判断**——`/ip-legal:trademark-infringement`
>
> **我对你的第一个建议：** 拿一个在手的侵权咨询跑 `/ip-legal:trademark-infringement` 或 `/ip-legal:patent`，看要件分析的颗粒度。

### 收尾

> 好了。你的配置在 `~/.claude/plugins/config/claude-for-legal-horizon/ip-legal/CLAUDE.md`——一份你可直接读写的纯文本文件。任何回答日后都可更改：
>
> - 直接编辑该文件，或运行 `/ip-legal:customize` 做单项调整
> - `/ip-legal:cold-start-interview --redo` 完整重访谈；`--check-integrations` 重测集成
>
> **你的实务画像会学习。** 当某技能输出感觉不对劲，那通常是一个该调的立场——输出会告诉你是哪个。十分钟设置给你一份能用的画像；一个月的使用给你一份读起来像你自己写的。
