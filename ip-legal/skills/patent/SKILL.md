# 专利分析

系统化拆解专利文件，围绕权利要求确定保护范围，并支撑侵权、稳定性与价值层面的技术判断。所有产出均为**初步技术分析**，不构成法律意见。

## 工作流程

### 第一步：明确分析目标

先确认本次属于哪一类任务，不同目标决定了分析的侧重：

- **技术要点提取**：快速读懂单件专利保护什么。
- **多专利比对**：评估专利组合，比较保护范围的宽窄与重叠。
- **侵权比对**：将目标产品/方法与权利要求逐一对照。
- **稳定性分析**：评估专利被宣告无效的风险。
- **FTO 分析**：产品上市前，排查是否落入他人在先专利。
- **价值评估**：为许可、转让、融资或质押提供参考。
- **规避设计**：寻找绕开权利要求的技术路径。

### 第二步：识别专利文件结构

| 部分 | 分析用途 |
|------|----------|
| 扉页/著录项 | 专利号、申请日、专利权人、分类号，确定权利归属与期限起点 |
| 权利要求书 | **保护范围的法律界定，是全部分析的核心** |
| 说明书 | 解释权利要求中的术语含义，理解发明要解决的问题与创新点 |
| 摘要/附图 | 辅助理解，**不单独作为确定保护范围的依据** |

阅读权利要求的提示：

- 独立权利要求通常以"一种……"起头，定义**最大保护范围**。
- 从属权利要求多以"根据权利要求 X 所述……"起头，在其引用的权利要求基础上**附加**技术特征，范围更窄。
- 关注"其特征在于"之后的限定，区分前序部分与特征部分。

### 第三步：解析权利要求（内联要点）

1. **判断类型**：产品权利要求还是方法权利要求，二者技术特征的表达方式不同。
2. **拆分技术特征**：把一项权利要求分解为若干相对独立、可单独比对的技术特征单元，逐一编号。
3. **确定术语含义**：说明书中有明确定义的，按说明书定义；没有定义的，按所属技术领域普通技术人员的通常理解。
4. **划定保护范围**：以权利要求文字记载的内容为准，结合说明书及附图理解，注意限定词（如方位、连接关系、运动方式、数量、材料）对范围的收窄作用。

### 第四步：侵权比对（字面 + 等同，内联要点）

侵权判断应当落在**独立权利要求**上，按下列顺序进行：

1. **锁定独立权利要求**：通常先看范围最大的独立权利要求，将其拆分为全部技术特征。
2. **逐一比对（字面侵权）**：对照被控对象，逐个技术特征核对是否被覆盖。
   - 全部技术特征被覆盖 → 字面侵权初步成立。
   - 存在任一技术特征缺失 → 字面侵权不成立，转入等同分析。
3. **等同分析**：对未被字面覆盖的特征，考察被控对象的对应特征是否以**基本相同的手段、实现基本相同的功能、达到基本相同的效果**，且为所属领域普通技术人员无需创造性劳动即可联想到的替换。需同时留意可能限制等同适用的因素（如审查或无效程序中的意见陈述、捐献规则、已有技术）。
4. **从属权利要求的作用**：只有在被控对象已落入独立权利要求后，才进一步比对从属权利要求；落入的从属权利要求越多，侵权认定越稳固，但**不得用从属权利要求的附加特征去否定本已成立的侵权**。

下面的判断链条概括了上述顺序：

```
拆分独立权利要求的全部技术特征
        │
   逐一字面比对：是否全部覆盖？
   ├─ 是 → 字面侵权初步成立 → 再看落入哪些从属权利要求（评估稳定性）
   └─ 否 → 对缺失特征做等同分析
              ├─ 构成等同 → 侵权初步成立
              └─ 不构成等同 → 不侵权
```

### 第五步：稳定性与价值评估（按需）

- **稳定性**：权利要求范围越大，越可能被现有技术覆盖而面临无效风险；从属权利要求可作为"后备"，必要时将其附加特征并入以维持有效。评估时关注新颖性、创造性、公开是否充分、权利要求是否得到说明书支持。
- **价值**：综合保护范围宽窄、稳定性高低、技术所处生命周期、市场覆盖与可规避难度等因素，给出相对量级的判断而非绝对数值。
- **FTO**：以目标产品的技术方案为基准，反向排查是否落入他人在先有效专利的权利要求，对命中的专利再做侵权比对与稳定性评估。

## 关键区分：独立权利要求 vs 从属权利要求

这是侵权分析中最易出错之处，务必区分：

- **独立权利要求**界定最大保护范围，是侵权判断的**唯一基准**。被控对象只要覆盖其全部技术特征即落入保护范围。
- **从属权利要求**在所引用权利要求之上附加技术特征，范围更小。它的意义在于**增强稳定性**——落入的从属权利要求越多，即便独立权利要求被限缩或无效，侵权认定仍可能维持。

常见错误对照：

| 易犯错误 | 正确做法 |
|----------|----------|
| 只看摘要或附图就下结论 | 以权利要求文字为准确定保护范围 |
| 把从属权利要求的附加特征当作侵权必要条件 | 侵权判断只看独立权利要求的全部特征 |
| 忽略限定词（方位、连接、运动、数量等） | 限定词直接收窄保护范围，逐字审视 |
| 用具体实施例代替权利要求 | 实施例仅为举例，范围以权利要求为准 |
| 跳过特征拆分直接给侵权结论 | 先逐特征拆分，再逐项比对 |
| 字面不覆盖即认定不侵权 | 字面不覆盖时仍须做等同分析 |
| 忽略说明书对术语的解释 | 术语含义优先按说明书定义理解 |

## 输入与输出

**输入**：专利文件（PDF / Markdown / 纯文本）；可选的目标产品或方法描述、现有技术资料、关注重点。

**输出**：结构化分析报告，视任务包含——技术特征拆分表、保护范围说明、侵权比对表（逐特征、字面/等同结论）、稳定性或价值评估、FTO 命中清单与处置建议。

## 注意事项

1. 本技能仅提供**初步技术分析**，不构成任何法律意见，重大决策前应咨询有资质的专业人士。
2. 实际侵权判定还需考量等同原则的边界、禁止反悔、现有技术抗辩、专利有效性等因素。
3. **专利法律状态（有效 / 失效 / 已被无效 / 权属变动）必须另行核实**，不得凭文件表面信息假定。
4. 出具任何对外文书前，必须逐件核对专利权人名称与登记信息完全一致；同一组合内不同专利可能权属不同。
5. 涉及具体法条或审查规则时，**必须核实条文现行有效文本，严禁凭记忆捏造条号或内容**。
6. 不同法域的专利制度存在差异，本分析以中国专利语境为默认前提。
