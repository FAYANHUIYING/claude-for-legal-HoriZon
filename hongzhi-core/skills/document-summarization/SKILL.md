# 法律文书结构化摘要

## 一、定位与原则

本技能面向各类中国法语境下的法律文书，输出客观、可检索、可对比的结构化摘要。核心动作有四步：识别文书类型 → 抽取关键要素 → 梳理逻辑链条 → 按类型排版输出。

四条贯穿始终的原则：

1. **忠于原文**——只整理文书已载明的内容，不补充原文没有的事实、观点或结论。
2. **客观中立**——不写带评价色彩的措辞（如"判得公正""处罚得当""一方明显恶意"），仅陈述事实与结论。
3. **凸显核心**——提取要素后用概括语言重组，删去与结论无关的细节和程序套话，不做逐句缩写。
4. **区分类型**——不同文书的功能和信息结构不同，摘要重点须随之调整。

> **免责说明：** 本摘要仅反映对应文书所记载的内容，不构成对案件实体问题的法律意见，亦不替代专业法律判断；扫描件经识别可能存在文字或数字偏差，关键信息请以原件为准。

## 二、文书类型与摘要重点

先判型、再选模板；切忌套着固定格式去硬塞内容，否则极易错过该类文书真正的要害。

| 文书类型 | 主要功能 | 摘要侧重 | 与他类的关键区别 |
|----------|----------|----------|------------------|
| 判决书 | 处理实体争议 | 事实认定、法律适用、裁判理由、判项 | 突出文书对事实的"认定"，区别于当事人"主张" |
| 裁定书 | 处理程序事项 | 程序阶段、申请或异议内容、所依规范、程序结论 | 不解决实体争议，只处理程序问题 |
| 调解书 | 确认双方合意 | 争议事项、调解协议条款、履行安排 | 体现自愿合意，非裁判机关单方裁断 |
| 仲裁裁决书 | 作出终局裁决 | 仲裁请求、事实认定、仲裁庭意见、裁决主文 | 一裁终局，以仲裁协议为基础 |
| 行政处罚决定书 | 实施行政制裁 | 违法事实、处罚依据、处罚种类与幅度、救济途径 | 行政机关单方作出，可申请复议或提起诉讼 |

判型可结合三项线索：标题关键词（如"民事判决书""执行裁定书""仲裁裁决书""行政处罚决定书"）、文书功能、作出主体（法院／仲裁机构／行政机关）。

## 三、六要素框架

无论何种文书，摘要均围绕以下六个要素组织：

1. **案件信息**——谁、何时、何地、由何机构作出（名称、案号或文号、日期、作出主体、当事人及其诉讼地位）。
2. **事实或程序背景**——发生了什么，仅保留与结论直接相关的部分。
3. **争议焦点**——文书真正要解决的核心法律问题。
4. **法律依据**——援引了哪些规范。
5. **结论结果**——最终如何处理。
6. **说理过程**——如何由事实加规范推导出结论。

## 四、分步工作流

### 步骤 1　识别文书类型
依据标题、功能、作出主体确定类型，并据此选择第六部分对应模板。

### 步骤 2　抽取案件信息
从文书首部与尾部提取完整的引用信息（名称、案号／文号、日期、作出机构、当事人）。此部分尽量沿用原文表述，以便后续检索与引用。

### 步骤 3　提炼事实或程序背景
- 判决书／仲裁裁决书／行政处罚决定书：提炼与实体结论相关的事实，**严格区分"当事人主张的事实"与"文书认定的事实"**，只留影响结论的关键事实。
- 裁定书：提炼程序阶段与申请／异议内容，不涉实体争议。
- 调解书：提炼争议事项、主要分歧点及促成合意的背景。

### 步骤 4　归纳争议焦点
把争议表述为一个或多个明确、可作答的法律问题，紧扣文书实际解决的问题，避免泛化为背景描述。

| 不可取（过于宽泛） | 可取（具体法律问题） |
|--------------------|----------------------|
| "双方产生了纠纷" | "被告抓取原告平台数据的行为是否构成不正当竞争" |
| "存在争议" | "被告提出的管辖权异议是否成立" |
| "赔偿的问题" | "原告主张的赔偿数额是否有事实与法律依据" |

### 步骤 5　提取结论或处理结果
准确概括最终处理。判决用"支持／驳回／确认／变更"，裁定用"准许／驳回／移送／中止／终结"，调解用"达成如下协议"，仲裁用"裁决……"，行政处罚用"决定给予……"。务必把"当事人主张"与"文书结论"分清，不能把请求误写成判项。

### 步骤 6　提取法律依据
逐条列出被援引并作为裁判、裁决或处罚依据的规范：法律、司法解释、行政法规或规章、合同条款、仲裁规则、行政裁量基准等。"参照／参考"的规范单独列示；若条文已失效或经修订，可在摘要中注明。

### 步骤 7　概括说理过程
用精炼语言呈现"采信哪些事实 → 如何适用规范 → 如何回应主要抗辩 → 如何得出结论"的因果链条，不复述完整诉辩经过，不做空泛表述（如"依法作出判决"）。

### 步骤 8　按类型排版输出
依据已识别的文书类型，选用第六部分的对应模板成稿。

## 五、常见偏差与纠正

| 偏差 | 表现 | 纠正 |
|------|------|------|
| 判型错误 | 把裁定书当判决书摘，大段铺陈实体争议 | 先判型再选模板 |
| 主张与认定混淆 | 把当事人主张写成"文书认定的事实" | 严格区分二者，认定须标明出处 |
| 焦点空泛 | "双方有争议""存在纠纷" | 表述为可作答的法律问题 |
| 掺入评价 | "公正地认为""处罚十分合理" | 删去评价性形容词，只陈述事实与结论 |
| 复述非摘要 | 大段照抄原文仅作删减 | 抽取要素后用概括语言重组 |
| 依据笼统 | 仅写"根据相关法律规定" | 列出具体法律名称加条文序号 |
| 说理空洞 | "依法作出处理" | 呈现"事实→规范→结论"的完整链条 |
| 套话冗余 | 保留送达、开庭通知、合议庭组成等 | 压缩或省略对结论无影响的程序内容 |

## 六、输出模板

### 模板 A·通用六要素（适用任何文书）

```markdown
## 文书摘要

### 一、案件信息
- 名称：[…]　- 案号／文号：[…]　- 日期：[…]
- 作出机构：[…]　- 当事人：[原告／申请人／被处罚人] 对 [被告／被申请人／处罚机关]

### 二、事实或程序背景
[第三人称、仅保留与结论直接相关的要点]

### 三、争议焦点
1. [表述为法律问题]

### 四、结论
- [对各焦点的处理结果]

### 五、法律依据
- 《XX法》第X条；[司法解释／法规／合同条款／仲裁规则等]

### 六、说理过程
[从事实认定到法律适用再到结论的因果链条]
```


## 七、特殊情形

- **多原告／多被告**：在当事人部分列清各方诉讼地位，事实与结果分别对应，避免混同。
- **二审／再审**：在程序背景说明审级，重点提炼改判或维持的差异点及其理由（再审则提炼申请再审理由与审查结论）。
- **关联或系列案件**：开头标注与关联案件的关系；批量诉讼可简化共同事实、突出个案差异。
- **扫描件识别**：识别不确定处以 `[？]` 标注，金额、日期、案号等关键信息尤需提示核对原件。
- **涉外或涉港澳台**：注意识别准据法与冲突规范，域外送达、取证等程序问题在裁定中重点提取。
- **敏感信息**：涉及不公开审理、商业秘密或个人隐私的，遵守披露限制，必要时对敏感信息脱敏。

## 八、成稿核对清单

```markdown
□ 文书类型是否已准确识别并选用对应模板？
□ 案件信息是否完整（名称／案号／日期／机构／当事人）？
□ 事实是否只保留了与结论直接相关的信息？
□ 是否严格区分了"当事人主张"与"文书认定"？
□ 争议焦点是否提炼为明确的法律问题？
□ 结论是否准确反映文书最终处理结果？
□ 法律依据是否列出了具体法条（名称＋序号）？
□ 说理是否呈现了"事实→规范→结论"的链条？
□ 是否未掺入任何主观评价性措辞？
□ 程序套话是否已适度压缩？
□ 扫描件识别的不确定处是否已标注？
```
