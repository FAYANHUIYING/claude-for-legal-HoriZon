# /cold-start-interview

1. 检查 `~/.claude/plugins/config/claude-for-legal-horizon/hongzhi-core/CLAUDE.md`。若带 `--check-integrations`，跳过访谈——只重跑第 0 部分的"接了什么？"检测，并在该配置路径重写 `## 可用集成` 表。探测时：只有当某 MCP 工具调用确实成功，才报告 ✓；已配置但未测试的连接器标 ⚪ 并附一行确认方法。**绝不仅凭 `.mcp.json` 声明就报告 ✓**——那会误导用户以为接好了，其实没有。
2. 运行下面的访谈：先第 0 部分（角色 + 执业设置 + 集成），再领域专属问题。
3. 索取种子文件（见下）。
4. 写入 `~/.claude/plugins/config/claude-for-legal-horizon/hongzhi-core/CLAUDE.md`，按需创建上级目录。

## 冷启动检查

读取 `~/.claude/plugins/config/claude-for-legal-horizon/hongzhi-core/CLAUDE.md`：
- **不存在** → 开始访谈。
- **包含 `<!-- SETUP PAUSED AT: -->`** → 向用户问好，提议从该小节继续。
- **包含 `[PLACEHOLDER]` 标记但无暂停注释** → 模板从未完成；提议重新开始，或从占位符起始处继续。
- **已填充（无占位符、无暂停注释）** → 已配置；除非带 `--redo`，否则跳过。

模板结构在 `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md`——以它作为分节脚手架。把填好的实务画像写入配置路径，按需创建上级目录。

## 检查共享的所务画像

查找 `~/.claude/plugins/config/claude-for-legal-horizon/company-profile.md`。

- **若存在：** 读取它。一行确认："你是 [姓名]，[执业设置]，就职于 [律所/公司]，在 [管辖地] 执业。对吗？（或说'更新'来修改共享画像。）" 确认后跳过所务问题，直接进入本插件专属问题。
- **若不存在：** 你将是该用户配置的第一个插件。问所务问题（所名/业务/规模/管辖地/风险姿态/上报联系人），按本插件 `references/company-profile-template.md` 模板写入共享画像，并告诉用户："我已保存你的所务画像——其他法律插件会读取它并跳过这些问题。"

## 访谈开始前

以"先分叉"的开场白起头，3-4 行短句，先问速启还是完整：

> **`hongzhi-core` 是全所底座**——术语规范、法律后果推导、证据三性评估、证据链映射、风险优先级、法律分析、文书摘要、严谨作答，被各业务插件反复调用。它通常是你安装的第一个插件，所以本访谈还负责建立全所共享的所务画像。
>
> **2 分钟**：完成你的角色、执业设置、共享所务画像（所名/业务/管辖地/风险姿态）和集成检测。 **15 分钟**：再加上输出行文偏好（备忘录 vs 要点式）、风险量表校准、术语口径偏好（如「解除」vs「辞退」的措辞习惯）。
>
> 速启还是完整？（随时可用 `/hongzhi-core:cold-start-interview --redo` 升级。）

**速启路径：** 只问第 0 部分和最关键的 1-2 个领域问题，其余以 `[DEFAULT]` 标记写入配置，并收尾说明：当某技能输出感觉不对劲时，通常就是该调的某个默认值——它会告诉你是哪个。

**完整路径：** 走完下面全部小节。

设置只从你的回答和你明确上传的文档构建画像——不读个人 Claude 历史、其他对话或主目录下的 CLAUDE.md。若对话上下文中已可见相关信息（如先前提到的所名），先问"要用它吗？"再使用。

## 访谈节奏

- **假设答案已经写在某处。** 凡超过一句话的内容，默认问法是"贴个链接或文档，或给我简短版"——别逼用户把已写好的东西重打一遍。
- **一轮不超过 2-3 个可作答的提问**（子项也算）。检验：用户能否不滚屏答完？
- **暂停与续接。** 开头告诉用户：说"暂停"即保存进度——写部分配置并在顶部加 `<!-- SETUP PAUSED AT: [小节名] -->`，未答字段标 `[PENDING]`。重跑时从停处继续，不重问已答的。
- **绝不写带静默缺口的配置。** 写入前回看：列出被跳过的问题，问"现在补几个，还是留作占位符？" 每个占位符都应是用户主动选择跳过的。
- **用户陈述的法律事实随到随核。** 用户给出的条号、期限、阈值、管辖地与你所知冲突时，写入配置前先说出来：「你说 X；我的理解是 Y——确认哪个写进画像？`[前提已标记—需核实]`」错误事实进了 CLAUDE.md 会传播进未来每份输出。

## 第 0 部分：谁在用，以及接了什么

#### 谁在用？

> 谁会日常使用本插件？（这决定每份输出的工作成果页眉。）
>
> 1. **律师/法律专业人员** → 页眉 `保密 — 律师工作成果 — 宏志律师事务所 · 依律师指导制作`
> 2. **非律师但有律师对接** 3. **非律师且无对接** → 页眉 `研究笔记 — 非法律意见 — 据以行事前须经执业律师审查`

若答案是 2 或 3，说一次：输出将表述为供律师审查的研究而非定论；在有法律后果的步骤（签署、发函、立案、回复机关）前会暂停并提示律师审查。这不是免责声明，是工具擅长什么（检索/组织/结构）与持牌律师对具体情形的判断之间的分界。

#### 执业设置

> 独立执业/小所（无层级） | 中大型律所 | 企业内部法务 | 政府/法律援助/公益 | 都不是（用你自己的话描述，我据实建画像而不硬塞框）

独立/小所 → 上报问题改问"何时请外聘律师或同事给第二意见"；中大型所/内部法务 → 问审批链与签字人；公益 → 问督导结构。

#### 接了什么？

> 本插件可对接：法律检索（元典/北大法宝）、飞书/IM、文档存储。让我检查你配了哪些——需要它们的功能会工作，没有的优雅回退到手工，而非静默失败。

**检查实际接通的，而非配置了的**：能测试就调一个简单 MCP 工具，成功才报 ✓；无从探测报 ⚪"已配置未验证"；未找到报 ✗ 并附接入方法与回退说明。日后接上了，重跑 `/hongzhi-core:cold-start-interview --check-integrations`。

把以上写入配置的 `## 使用者角色`、`## 可用集成` 小节。

## 领域专属问题

### 第 1 部分：输出与口径偏好

> 底座技能没有自己的业务立场——它们继承调用方业务插件的立场。这里只定全所统一的**呈现层**偏好：
>
> 1. **行文密度**：分析输出偏好完整备忘录体，还是要点式速览？（影响 legal-analysis、document-summarization 的默认输出形态。）
> 2. **风险量表校准**：🔴阻断/🟠高/🟡中/🟢低 四级量表，你所内有没有既有的对应口径（如「重大/较大/一般/提示」）？有就贴给我，我做映射。
> 3. **术语口径**：有没有所内统一的措辞习惯清单（如统一用「解除」不用「辞退」、统一用「相对方」不用「对方」）？有就贴上，legal-terminology 会以它为准。

## 种子文件

> 可选但有价值：贴 1-2 份你所定稿的分析备忘录或意见书。我从中提取行文风格（标题层级、引用格式、结论先行还是论证先行），写进画像供 8 个底座技能统一遵循。没有就说"跳过"，先用默认。

## 撰写实务画像

按 `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md` 的模板结构，把填好的画像写入 `~/.claude/plugins/config/claude-for-legal-horizon/hongzhi-core/CLAUDE.md`。凡基于少于足量文档构建的小节标 `[LIMITED DATA — 已审阅 N 份文档]`。

## 写完之后

**展示本插件能做什么**（具体清单，不是笼统模板）：

> - **结构化法律分析**——对一段事实做问题识别→依据→程序→风险的结构化分析。试：`/hongzhi-core:legal-analysis`
> - **证据三性评估**——逐份评真实性/合法性/关联性并给补强建议。试：`/hongzhi-core:evidence-evaluation`
> - **证据链映射**——把证据映射到待证事实，缺口一目了然。试：`/hongzhi-core:evidence-argument-chain`
> - **多风险排序**——按概率×影响把一堆风险点排出轻重缓急。试：`/hongzhi-core:risk-prioritization`
> - **文书摘要 / 术语规范 / 法律后果推导 / 严谨作答**——`/hongzhi-core:document-summarization` 等。
>
> **我对你的第一个建议：** 拿一份手头的判决书或长备忘录跑 `/hongzhi-core:document-summarization`，30 秒看出行文偏好对不对味。

### 收尾

> 好了。你的配置在 `~/.claude/plugins/config/claude-for-legal-horizon/hongzhi-core/CLAUDE.md`——一份你可直接读写的纯文本文件。任何回答日后都可更改：
>
> - 直接编辑该文件，或运行 `/hongzhi-core:customize` 做单项调整
> - `/hongzhi-core:cold-start-interview --redo` 完整重访谈；`--check-integrations` 重测集成
>
> **你的实务画像会学习。** 当某技能输出感觉不对劲，那通常是一个该调的立场——输出会告诉你是哪个。十分钟设置给你一份能用的画像；一个月的使用给你一份读起来像你自己写的。
