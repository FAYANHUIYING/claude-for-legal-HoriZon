# 尽职调查清单定制生成

## 用途

为股权投资、并购或重大合作交易中的尽职调查环节，生成一份贴合标的实际情况的清单。技能不输出固定模板，而是先理解交易要素，再按行业、阶段、交易结构动态调整清单的深度与侧重，最终交付六大维度清单 + 信息请求清单（IRL）+ 排期建议 + Red Flag 检查点。

核心定位：把"通用尽调框架"裁剪成"这一单的尽调清单"。

## 触发条件

出现以下任一情形即按本流程执行：用户要求生成尽调清单、DD checklist、信息请求清单（IRL）；要求做尽调排期或时间规划；要求梳理某行业/某阶段的尽调要点；要求识别交易中的 Red Flag。

## 工作流

### 第一步 · 锁定交易要素

先把以下要素问清楚，缺失即主动追问，不要在信息不全时套用模板：

| 要素 | 作用 |
|------|------|
| 标的所属行业/细分赛道 | 决定加挂哪些行业专项清单 |
| 发展阶段（初创/成长/成熟） | 决定每个维度的查证深度 |
| 交易类型（少数股权/控股收购/整体收购/可转债等） | 决定侧重点与特殊关注条款 |
| 投资或交易规模 | 决定重要性阈值与抽查比例 |
| 尽调可用周期 | 决定排期压缩或铺开 |
| 已掌握材料（推介材料/数据室/财报等） | 决定 IRL 从哪里补缺 |

### 第二步 · 按"行业 × 阶段 × 交易类型"三轴定制

三轴叠加决定最终清单形态，三者相互修正：

- 按阶段调深度：初创期侧重团队背景、市场假设、产品验证与核心知识产权归属；成长期侧重单位经济模型、增长可持续性、竞争壁垒与现金消耗速度；成熟期/整体收购侧重稳定现金流、运营效率、负债承受力与交割后整合可行性。
- 按交易类型调侧重：少数股权侧重信息知情权、否决权、反稀释与领售/随售等保护性条款；控股或整体收购侧重管理层留任、过渡安排、或有负债承接与整合落地。
- 按行业加挂专项：在通用六大维度之上，针对硬科技/制造类补产线、供应链、核心专利与关键设备；针对软件/平台类补数据合规、续费与留存、系统架构与开源合规；针对消费/医疗/能源/金融等补各自的牌照资质、准入门槛与监管口径。无现成专项时，从通用清单出发，按上述逻辑临时补充行业要点并注明依据。

### 第三步 · 生成六大维度清单

逐项标注：所属维度与子类、具体查证事项、优先级（必查/建议/可选）、责任方、预计耗时。要点见下文「六大维度清单」。

### 第四步 · 由清单反推信息请求清单（IRL）

把每条查证事项翻译成一份具体的文档或数据请求，按维度归类，标注期望格式（导出表格/原件扫描/系统导出）与提供优先级：P0 首批必交（股权结构、核心财务、关键合同、资质证照）；P1 次批补充（客户/供应商名单、技术架构、用工与社保明细）；P2 视情补充（历史数据、行业报告、内部分析）。

### 第五步 · 制定尽调排期

依据可用周期与关键节点（投决会、排他期截止）铺排阶段，并按实际周期压缩或扩展。一个典型四周节奏：

- 第 1 周：数据室开放、发出 IRL、审阅 P0 材料、搭建问题清单。
- 第 2 周：管理层访谈、关键客户与供应商访谈、财务模型初搭。
- 第 3 周：深度访谈、实地走访、必要的第三方专项尽调。
- 第 4 周：汇总 Red Flag、调整估值与交易条款假设、形成尽调备忘。

### 第六步 · Red Flag 扫描与分级

逐维度扫描异常信号（见下文「Red Flag 检查点」），并三级分类：

- 红色（交易破坏者）：需立即查清，可能触发交易终止或重大条款调整。
- 黄色（重大风险）：需深入核查，可能影响估值或交易条款。
- 绿色（一般关注）：持续跟踪，通常可通过条款安排或对赌机制覆盖。

## 六大维度清单（要点内联）

以下为各维度的核心查证事项，作为定制起点；实际清单按第二步三轴展开为更细的子项。

### 商业
- 市场规模、增速与标的真实份额，及测算口径是否可验证。
- 客户结构与集中度，前几大客户的贡献占比与可替代性。
- 竞争格局、进入壁垒与标的的差异化优势是否可持续。
- 商业模式与定价机制，关键收入假设的合理性。

### 财务
- 历史财务的真实性与可比性，收入确认口径与异常波动。
- 盈利质量：毛利结构、费用弹性、一次性损益与现金流匹配度。
- 资产负债质量：应收账款账龄、存货跌价、表外负债与或有负债。
- 关联交易与资金往来是否公允、是否存在利益输送或资金占用。
- 税务合规：欠缴、优惠依据、历史涉税风险。

### 法律
- 主体资格、股权结构与历史出资、代持及权属瑕疵。
- 重大合同的有效性、控制权变更与提前终止条款。
- 资质牌照、行政许可的齐备与有效性。
- 知识产权归属、权属链条与许可瑕疵。
- 诉讼仲裁、行政处罚与潜在合规风险。
- 担保、抵质押、对外投资等或有事项。

### 技术
- 核心技术来源、自研比例与权属清晰度。
- 系统架构稳定性、可扩展性与单点故障风险。
- 数据合规与信息安全：数据来源、留存、跨境与个人信息处理。
- 技术债务、关键人员依赖与研发可持续性。

### 运营
- 供应链稳定性与对关键供应商/单一来源的依赖。
- 产能、交付能力与质量管控体系。
- 关键业务流程的内控有效性与可复制性。
- 重大经营事故、退换货与售后纠纷历史。

### 人力
- 核心团队稳定性、竞业与保密安排的有效性。
- 用工合规：劳动合同、社保公积金、加班与工时。
- 劳动争议、群体性事件与未决纠纷。
- 股权激励的设计、行权安排与潜在争议。

## Red Flag 检查点（内联）

逐维度优先扫描以下高信号项，命中即按三级分类标注：

- 商业：单一客户收入占比过高、客户加速流失、市场份额持续下滑、关键假设无法验证。
- 财务：现金消耗加速且无明确收敛路径、应收账款异常膨胀、关联交易金额异常、毛利与同行严重背离。
- 法律：重大未决诉讼或仲裁、核心知识产权权属存疑、关键资质缺失或临近失效、重大未披露担保。
- 技术：关键技术依赖外部授权、数据合规存在系统性缺口、架构存在单点故障、研发高度依赖个别人员。
- 运营：供应链高度依赖单一来源、产能存在硬瓶颈、近期发生重大质量或安全事故。
- 人力：核心人员近期密集离职、存在群体性劳动争议、社保用工历史欠账较大、股权激励安排存在争议。

## 输出与呈现

- 清单条目较少时，可直接以分级表格在对话中呈现。
- 当任一清单超过约 10 行，或需要同时交付清单 + IRL + 排期 + Red Flag 多张表时，建议导出为带多个工作表的电子表格，或生成结构化仪表盘视图，便于过滤、分配责任方与跟踪进度。建议的工作表划分：六大维度清单、Red Flag 汇总、信息请求清单（IRL）、尽调排期、行业差异对照。
- 表格可用底色区分优先级（必查/建议/可选）与 Red Flag 级别，并冻结表头、开启筛选。

## 免责声明

本技能输出为尽职调查的工作辅助清单，用于组织查证范围与进度，不构成法律、财务、税务或投资意见，也不替代实地核查、外部审计、专业鉴定与正式专项意见。具体交易中的范围取舍与结论，应由相应专业人员结合实际材料独立判断。
